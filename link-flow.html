aiCodingLight · Dataflow

从二维码到手机通知的完整链路

一个配对码（ntfy topic）串起两端。配对是一次性的把码同步过去；之后每次 AI 工具变状态，都沿同一条路推到手机。两端只经服务器中转，从不直连。

电脑 Mac 手机 Android ntfy 服务器 红 · 需确认 黄 · 工作中 绿 · 完成

配对阶段

一次性 · 电脑出码 → 手机扫码

03电脑手机

两端各存同一个配对码

~/.config/aicodinglight/config  ·  Prefs

配对完成的标志：两端持有同一个 topic。二维码只是把它从电脑单向送到手机，此后互不感知对方地址。

电脑 config
NTFY_TOPIC=aicl-xxx
NTFY_SERVER=https://ntfy.sh
TRANSPORT=none   # 纯手机模式，跳过串口灯

04电脑

AI 工具触发事件，调用各自配置里的命令 set-light.sh <颜色>。脚本顺手从 hook 的 stdin JSON 读 session_id，拼出标签用于区分多个窗口。

标签优先级

05电脑

组装并推送到服务器

core/notify-phone.sh

读 config 的 topic/server，按颜色定 body 词与标题，curl POST 到 SERVER/TOPIC（2s 超时、后台、静默失败）。body 传语义、Title 传身份。

06手机

收到 · 弹通知 + 存历史

NtfyService.handle() → Notifier · HistoryStore

SSE 收到 message，解析出 body 词与 title。按 body 定颜色，用 title 展示，通知 id = 标签的 hash——不同窗口并存、同窗口红→绿覆盖；同时写入 SQLite 供按日/项目统计。

收到的 ntfy 消息
{ "event":"message",
  "message":"attention",
  "title":"🔴 需要你确认 · aiCodingLight#78c902" }

两条设计主线

A

一个 topic 串起两端。 配对即同步这个配对码；之后全部走 ntfy 服务器中转，手机与电脑互不知道对方地址，也无需同一局域网。

B

body 传语义，Title 传身份。 body 永远是 attention/done/working，让 App 稳定判色；session 标签走 Title，App 再用它给通知分组。改标签不影响判色。
